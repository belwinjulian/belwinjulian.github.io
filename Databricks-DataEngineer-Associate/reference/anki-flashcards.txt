What problem does the Lakehouse architecture solve?	Data warehouses were reliable but only handled structured data. Data lakes were flexible but had no ACID guarantees. The Lakehouse gives you <b>both</b> — reliability of a warehouse + flexibility of a lake — on top of cheap cloud storage.	L01 architecture
What are the two things Delta Lake stores on disk?	<b>1. Parquet files</b> — the actual data<br><b>2. _delta_log/</b> — a folder of JSON transaction log files, one per write operation	L01 L02 delta-lake
What does the <code>_delta_log/</code> folder do?	It records every write (INSERT, UPDATE, DELETE, MERGE) as a JSON file. This log is what gives Delta Lake its ACID properties and enables Time Travel.	L01 L02 delta-lake
What does ACID stand for (plain language)?	<b>A</b>tomicity — write fully succeeds or fully fails, no partials<br><b>C</b>onsistency — data always matches schema rules<br><b>I</b>solation — concurrent writes don't corrupt each other<br><b>D</b>urability — committed writes survive crashes	L01 delta-lake
What layer does data go to in the Medallion Architecture if it is raw and untouched?	<b>Bronze</b> — raw data exactly as it arrived, no changes, full audit trail	L01 medallion
What layer does data go to if it has been cleaned, deduplicated, and validated?	<b>Silver</b> — cleaned, deduplicated, validated, schema enforced	L01 medallion
What layer does data go to if it is aggregated and ready for dashboards or reports?	<b>Gold</b> — aggregated, business-ready, used by dashboards, reports, and ML models	L01 medallion
What is a Databricks Cluster?	A group of virtual machines that run your Spark code. You spin them up to compute and shut them down to save money.<br><br><b>All-Purpose</b> = interactive use while working<br><b>Job cluster</b> = automated production runs	L01 platform
What is Unity Catalog?	Databricks' governance layer — controls who can read/write which tables and tracks data lineage.	L01 governance
What SQL command shows all past versions of a Delta table?	<code>DESCRIBE HISTORY table_name;</code><br><br>Shows version number, timestamp, operation type (WRITE/UPDATE/DELETE/MERGE), and who made the change.	L02 time-travel
How do you query a Delta table as it was at version 3?	<code>SELECT * FROM orders VERSION AS OF 3;</code>	L02 time-travel
How do you query a Delta table as it was on a specific date?	<code>SELECT * FROM orders TIMESTAMP AS OF '2026-05-01';</code>	L02 time-travel
What is the difference between <code>SELECT ... VERSION AS OF</code> and <code>RESTORE TABLE</code>?	<code>SELECT ... VERSION AS OF</code> = read-only, the table is unchanged<br><br><code>RESTORE TABLE orders TO VERSION AS OF 2</code> = actually rolls the table back to that version	L02 time-travel
What does <code>OPTIMIZE</code> do?	Compacts many small Parquet files into fewer, larger files — makes queries faster. Does <b>not</b> delete data, just reorganises files.	L02 optimize
What does <code>ZORDER BY</code> do (used with OPTIMIZE)?	Co-locates rows with the same column values on disk, speeding up queries that filter on that column.<br><br><code>OPTIMIZE orders ZORDER BY (customer_id);</code>	L02 optimize
What does <code>VACUUM</code> do?	Permanently and irreversibly deletes old Parquet files that are no longer referenced by the current table version.	L02 vacuum
What is VACUUM's default retention period?	<b>7 days (168 hours)</b>. Files newer than 7 days are kept so Time Travel still works within that window.	L02 vacuum
What happens to Time Travel if you run <code>VACUUM orders RETAIN 24 HOURS</code>?	Time Travel is cut off for any version older than 24 hours. Those versions are gone permanently.	L02 vacuum time-travel
What is the difference between a managed table and an external table when you DROP it?	<b>Managed:</b> <code>DROP TABLE</code> deletes the data files permanently<br><b>External:</b> <code>DROP TABLE</code> removes only the metadata; data files in S3/ADLS survive	L02 tables
How do you create an external Delta table?	<code>CREATE TABLE orders_external<br>LOCATION 's3://my-bucket/orders/';</code><br><br>You specify the file path. Databricks does not own or delete those files.	L02 tables
What does <code>COPY INTO</code> do?	Loads files from cloud storage into a Delta table using SQL. It is <b>idempotent</b> — it tracks which files it has already loaded and skips them on re-runs.	L03 ingestion copy-into
What does "idempotent" mean for COPY INTO?	Running <code>COPY INTO</code> multiple times on the same folder is safe — it only loads each file <b>once</b> and skips already-processed files.	L03 ingestion copy-into
Write the basic COPY INTO syntax.	<code>COPY INTO orders<br>FROM 's3://my-bucket/raw/'<br>FILEFORMAT = CSV<br>FORMAT_OPTIONS ('header' = 'true', 'inferSchema' = 'true')<br>COPY_OPTIONS ('mergeSchema' = 'true');</code>	L03 ingestion copy-into
When should you choose COPY INTO over Auto Loader?	When loading a <b>bounded, known set of files</b> (e.g. a one-time historical load) and you want <b>SQL-only</b> — no Python. Best for hundreds to low thousands of files.	L03 ingestion copy-into
What keyword in code identifies Auto Loader?	<code>format("cloudFiles")</code> — this string is the identifier. Any code using it is Auto Loader.	L03 ingestion auto-loader
What does <code>checkpointLocation</code> do in Auto Loader?	Tracks which files Auto Loader has already processed. If the job restarts, it reads the checkpoint and continues from where it left off — no duplicate processing, no missed files.	L03 ingestion auto-loader
What happens if Auto Loader's checkpointLocation is deleted?	It reprocesses all files from the beginning — duplicates will occur.	L03 ingestion auto-loader
What does <code>cloudFiles.schemaLocation</code> do?	Stores the inferred schema for incoming files so Auto Loader doesn't re-infer it on every run. Also enables schema evolution tracking.	L03 ingestion auto-loader
What does <code>trigger(availableNow=True)</code> do?	Processes all files currently in the source folder, then stops. Acts like a batch run using a streaming API. Preferred over <code>trigger(once=True)</code>.	L03 L04 streaming triggers
What is the difference between <code>trigger(once=True)</code> and <code>trigger(availableNow=True)</code>?	<code>trigger(once=True)</code> runs a <b>single</b> micro-batch then stops.<br><br><code>trigger(availableNow=True)</code> runs <b>all</b> micro-batches until the source is fully caught up, then stops. <code>availableNow</code> is preferred.	L04 streaming triggers
When should you choose Auto Loader over COPY INTO?	When files arrive <b>continuously</b> over time, there are <b>millions of files</b>, or you need <b>automatic schema inference and evolution</b>.	L03 ingestion auto-loader
What is Structured Streaming's mental model?	It treats a stream as an <b>infinite table</b>. Each trigger, new rows arrive and are appended. Spark runs your query on the new rows only and writes results to the sink. The checkpoint tracks progress.	L04 streaming
What are the three output modes in Structured Streaming?	<b>append</b> — write new rows only (default for Delta)<br><b>complete</b> — rewrite the entire result table every trigger (aggregations)<br><b>update</b> — write only rows that changed since last trigger	L04 streaming output-modes
When do you use <code>outputMode("complete")</code>?	When you have a <b>running aggregation</b> and want the full result rewritten each trigger (e.g. a running total or COUNT by category).	L04 streaming output-modes
What does Delta Lake do by default when you write data with an extra column the table doesn't have?	<b>Rejects the write and throws an error.</b> Schema enforcement is on by default — Delta protects the table from mismatched data.	L02 L04 schema
What are two ways to enable schema evolution in Delta Lake?	<b>1. Per-write:</b> <code>.option("mergeSchema", "true")</code><br><b>2. Session-level:</b> <code>spark.conf.set("spark.databricks.delta.schema.autoMerge.enabled", "true")</code>	L02 L04 schema
What happens to existing rows when schema evolution adds a new column to a Delta table?	Existing rows get <b>NULL</b> for the new column. Delta never drops columns automatically.	L04 schema
What does schema evolution in Delta Lake NOT do?	It does <b>not</b> change the type of an existing column, delete columns missing from the source, or reorder columns.	L04 schema
What is Auto Loader's <code>cloudFiles.schemaEvolutionMode = "rescue"</code>?	Unexpected or mismatched columns are captured in a special <b><code>_rescued_data</code></b> JSON column instead of failing or being dropped.	L04 schema auto-loader
What is the <code>_rescued_data</code> column?	A JSON column Auto Loader creates when in <code>rescue</code> schema evolution mode. It stores any columns that didn't match the expected schema, instead of failing the stream or dropping the data.	L04 schema auto-loader
What are the four Auto Loader schema evolution modes?	<b>addNewColumns</b> (default) — adds new columns, stream restarts<br><b>rescue</b> — stores unexpected columns in <code>_rescued_data</code><br><b>failOnNewColumns</b> — throws error on new column<br><b>none</b> — silently drops new columns	L04 schema auto-loader
What is Lakeflow Connect?	Databricks' managed connector service for <b>SaaS sources</b> (Salesforce, Workday, Google Analytics, ServiceNow, etc.). Provides no-code incremental sync directly into Delta tables in Unity Catalog.	L04 ingestion lakeflow
When do you choose Lakeflow Connect vs Auto Loader?	<b>Lakeflow Connect</b> — source is a SaaS API (Salesforce, Workday, SAP…)<br><b>Auto Loader</b> — source is files in cloud storage (S3, ADLS, GCS)	L04 ingestion lakeflow
What does Lakeflow Connect handle automatically?	Authentication to the SaaS API, incremental sync (only changed records), schema mapping, error handling and retries, and scheduling via Lakeflow Jobs.	L04 ingestion lakeflow
A job fails halfway through writing to a Delta table. Is the table corrupted?	<b>No.</b> Delta Lake's Atomicity guarantee means the write either fully succeeds or fully fails — a failed mid-write leaves the table in its previous valid state.	L01 L02 delta-lake acid
A question asks which ingestion tool to use for a Salesforce data feed. What is the answer?	<b>Lakeflow Connect</b> — it is designed for SaaS API sources. Auto Loader is for files in cloud storage, not SaaS APIs.	L04 ingestion lakeflow
A team dropped a Delta table and lost all the data. Was it managed or external?	<b>Managed table.</b> Dropping a managed table permanently deletes the underlying data files. An external table's files would have survived the DROP.	L02 tables
What are the four join types and what does each keep?	<b>INNER</b> — matched rows only (unmatched dropped from both sides)<br><b>LEFT</b> — all left rows + matched right rows (unmatched right = NULL)<br><b>RIGHT</b> — all right rows + matched left rows (unmatched left = NULL)<br><b>FULL OUTER</b> — all rows from both sides (unmatched = NULL on missing side)	L06 joins
What is the default join type in SQL and PySpark?	<b>INNER</b> join. SQL: <code>JOIN</code> with no qualifier. PySpark: <code>how="inner"</code> is the default.	L06 joins
You want to keep all rows from the orders table even if there is no matching customer. Which join type do you use?	<b>LEFT join</b> — keeps all rows from the left (orders) table. Unmatched rows from the right (customers) come back as NULL.	L06 joins
What is the difference between <code>distinct()</code> and <code>dropDuplicates(["col"])</code>?	<code>distinct()</code> — removes rows where <b>every column</b> is identical (whole-row dedup)<br><code>dropDuplicates(["col"])</code> — removes rows where only the <b>specified column(s)</b> match; keeps one row per unique key	L06 dedup
Why can't you use <code>dropDuplicates(["order_id"])</code> to keep the most recent duplicate?	<code>dropDuplicates</code> is <b>non-deterministic</b> — it keeps a random row with no guarantee of which one survives. Use a window function with <code>row_number()</code> ordered by timestamp instead.	L06 dedup
How do you keep the most recent row per order_id when duplicates exist?	Use a window function:<br><code>w = Window.partitionBy("order_id").orderBy(col("updated_at").desc())<br>df.withColumn("rn", row_number().over(w)).filter("rn = 1").drop("rn")</code><br><br>Rank newest first, keep only rank 1.	L06 dedup window
What is the difference between <code>groupBy</code> and a window function in terms of output rows?	<code>groupBy</code> — collapses rows into one per group (you lose detail)<br>Window function — keeps <b>all original rows</b> and adds a computed column (rank, lag, etc.)	L06 window
What does <code>partitionBy("customer")</code> do inside a window function?	Restarts the window calculation for each unique customer. Each customer gets their own independent ranking/computation — like GROUP BY but without collapsing rows.	L06 window
What is the SQL clause execution order?	<b>FROM → WHERE → GROUP BY → HAVING → SELECT</b><br><br>SELECT runs last, which is why you can't reference a SELECT alias in WHERE or HAVING.	L06 sql aggregations
What is the difference between WHERE and HAVING?	<b>WHERE</b> — filters individual rows <em>before</em> grouping (cannot reference aggregated columns)<br><b>HAVING</b> — filters groups <em>after</em> aggregation (can reference COUNT, SUM, etc.)	L06 sql aggregations
This query fails — why?<br><code>SELECT customer, COUNT(*) AS order_count FROM orders WHERE order_count > 5 GROUP BY customer</code>	WHERE runs <b>before</b> GROUP BY and COUNT. At the point WHERE executes, <code>order_count</code> doesn't exist yet. Use <b>HAVING</b> instead: <code>GROUP BY customer HAVING COUNT(*) > 5</code>	L06 sql aggregations
What does MERGE INTO do?	Compares a source table to a target table on a key. <b>WHEN MATCHED</b> → UPDATE the existing row. <b>WHEN NOT MATCHED</b> → INSERT the new row. Performs both in a single operation.	L06 merge
What is the word for the pattern MERGE INTO performs?	<b>Upsert</b> — update if the row exists, insert if it does not. Any exam question using the word "upsert" → answer is MERGE INTO.	L06 merge
Fill in the blanks: <code>MERGE INTO target USING source ON target.id = source.id WHEN _____ THEN UPDATE ... WHEN _____ THEN INSERT ...</code>	<code>WHEN <b>MATCHED</b> THEN UPDATE</code> — row exists in both tables<br><code>WHEN <b>NOT MATCHED</b> THEN INSERT</code> — row only exists in source	L06 merge
A Silver table needs to be kept in sync with daily Bronze updates — existing records updated, new records inserted. What SQL command handles this?	<b>MERGE INTO</b> (upsert). WHEN MATCHED → UPDATE, WHEN NOT MATCHED → INSERT. This is the standard Bronze-to-Silver sync pattern.	L06 merge scenario
What is the difference between <code>rank()</code> and <code>dense_rank()</code> in a window function?	<code>rank()</code> — leaves gaps for ties: 1, 2, 2, <b>4</b>…<br><code>dense_rank()</code> — no gaps for ties: 1, 2, 2, <b>3</b>…<br><code>row_number()</code> — always unique, no ties: 1, 2, 3, 4…	L06 window
